# FJ-ker 详细使用说明

本文档说明如何从零开始使用 FJ-ker：硬件接线、PC 服务端安装、DashScope 密钥配置、固件烧录、日常使用、测试和常见问题排查。

FJ-ker 的工作方式是：

1. ESP32-S3-CAM 拍下题目照片。
2. 设备通过 Wi-Fi 把 JPEG 上传到 Windows PC 服务端。
3. PC 服务端调用 DashScope 的 Qwen 视觉模型解题。
4. PC 服务端把中文和 LaTeX 答案渲染成 384×168 横屏 1-bpp 页面位图。
5. ESP32-S3-CAM 下载页面位图并在 ST7305 单色屏上显示。

## 1. 使用前准备

### 1.1 你需要的硬件

- GOOUUU ESP32-S3-CAM 开发板，带 OV2640 摄像头和 PSRAM。
- YDP290H001 2.9 寸单色 SPI LCD，横屏使用，逻辑分辨率 384×168，ST7305 控制器。
- 5 个轻触按钮：拍摄、确认、取消、上一页、下一页。
- 1 个主电源滑动开关。
- TP4056 USB-C 充电模块。
- 602030 锂电池，约 250 mAh。
- TPS61088 升压模块，输出设置为 5.0 V。
- Windows PC，和设备连接到同一个 Wi-Fi 局域网。

### 1.2 你需要的软件

PC 服务端：

- Windows PowerShell。
- Python 3.11。当前 `server/install.ps1` 使用 `py -3.11` 创建虚拟环境。
- DashScope API Key。

固件烧录：

- VS Code。
- PlatformIO 扩展。
- 首次构建时需要网络，用于下载 PlatformIO 平台包和 ArduinoJson。

## 2. 硬件接线

### 2.1 电源接线

按下面方式连接电池、充电模块、开关、升压模块和 ESP32-S3-CAM：

```text
Battery(+) ─── TP4056 BAT+
Battery(-) ─── TP4056 BAT-
TP4056 OUT+ ─── 主电源开关 ─── TPS61088 VIN+
TP4056 OUT- ───────────────── TPS61088 VIN-
TPS61088 VOUT+ (5 V) ─── ESP32-S3-CAM 5V/VIN
TPS61088 VOUT- ─────── ESP32-S3-CAM GND
```

注意：

- TPS61088 输出必须先调到 5.0 V，再接 ESP32-S3-CAM。
- 显示屏 VCC 接 ESP32-S3-CAM 的 3V3，不要接 5 V。
- 主电源开关放在 TP4056 OUT+ 和 TPS61088 VIN+ 之间，这样关机时可以切断负载。
- TP4056 仍然直接连接电池，所以关机时仍可通过 TP4056 充电。

### 2.2 电池电压检测

电池电压通过 100 kΩ + 100 kΩ 分压接入 GPIO1：

```text
BAT+ ─[100 kΩ]─┬─[100 kΩ]─ GND
               └── GPIO1 / ADC
```

固件会在启动时读取电池电压。如果低于 3.3 V，会显示低电量提示并拒绝拍摄，以保护电池。

### 2.3 显示屏接线

| 显示屏信号 | ESP32 GPIO | 说明 |
|---|---:|---|
| SCLK | GPIO47 | SPI 时钟 |
| MOSI | GPIO21 | SPI 数据 |
| CS | GPIO42 | 低电平有效 |
| DC | GPIO41 | 数据/命令 |
| RST | GPIO40 | 复位 |
| VCC | 3V3 | 只能接 3.3 V |
| GND | GND | 共地 |

### 2.4 按钮接线

所有按钮都采用同一种接法：GPIO 一端接按钮，按钮另一端接 GND。固件会启用内部上拉。

建议每个按钮的 GPIO 到 GND 加一个 100 nF 电容，降低按键抖动和噪声。

| 按钮 | ESP32 GPIO |
|---|---:|
| 拍摄 Capture | GPIO45 |
| 确认 Confirm | GPIO48 |
| 取消 Cancel | GPIO39 |
| 上一页 PageUp | GPIO38 |
| 下一页 PageDown | GPIO2 |

注意：GPIO45 是启动绑定位。可以作为输入使用，但启动时不要被外部电路强制拉低。

## 3. 配置并启动 PC 服务端

PC 服务端负责接收照片、调用 Qwen、渲染答案页面，并把页面提供给设备下载。

### 3.1 安装依赖

打开 PowerShell，进入项目的 `server` 目录：

```powershell
cd E:\陈晨\FJ-ker\server
.\install.ps1
```

这个脚本会做三件事：

1. 如果不存在 `.venv`，使用 Python 3.11 创建虚拟环境。
2. 安装服务端依赖和测试依赖。
3. 安装 Playwright Chromium。

如果提示找不到 Python 3.11，请先安装 Python 3.11，并确认下面命令能运行：

```powershell
py -3.11 --version
```

### 3.2 配置 DashScope API Key

复制示例配置：

```powershell
copy .env.example .env
notepad .env
```

把 `.env` 改成类似下面这样：

```env
DASHSCOPE_API_KEY=sk-你的DashScope密钥
QWEN_MODEL=qwen3.7-plus
SERVER_HOST=0.0.0.0
SERVER_PORT=8080
MAX_PAGES=20
MAX_UPLOAD_BYTES=2000000
```

字段说明：

| 字段 | 说明 |
|---|---|
| `DASHSCOPE_API_KEY` | DashScope 控制台获取的 API Key，通常以 `sk-` 开头 |
| `QWEN_MODEL` | 默认使用 `qwen3.7-plus`；如果 DashScope 拒绝此模型名，可改成可用的视觉模型 |
| `SERVER_HOST` | 服务端配置项；默认保持 `0.0.0.0` |
| `SERVER_PORT` | 服务端配置项；默认 `8080` |
| `MAX_PAGES` | 最多返回多少页答案 |
| `MAX_UPLOAD_BYTES` | 允许上传的图片最大字节数，默认 2 MB |

不要把真实 `.env` 发给别人，也不要提交到 Git。

注意：当前 `run.ps1` 启动 Uvicorn 时，实际监听地址优先读取 PowerShell 环境变量；如果不设置环境变量，则默认监听 `0.0.0.0:8080`。如果你想临时改端口，例如改成 8081，请这样启动：

```powershell
cd E:\陈晨\FJ-ker\server
$env:SERVER_HOST = "0.0.0.0"
$env:SERVER_PORT = "8081"
.\run.ps1
```

### 3.3 启动服务端

在 `server` 目录运行：

```powershell
.\run.ps1
```

正常启动后，脚本会打印局域网 IPv4 地址，例如：

```text
局域网 IPv4 地址：
  http://192.168.1.42:8080
```

把这个地址记下来，后面要填入固件的 `SERVER_BASE_URL`。

### 3.4 检查服务端是否正常

在 PC 浏览器打开：

```text
http://你的PC局域网IP:8080/health
```

例如：

```text
http://192.168.1.42:8080/health
```

正常返回应类似：

```json
{"ok":true,"version":"0.1.0"}
```

如果 ESP32 无法访问服务端，请检查：

- PC 和 ESP32 是否在同一个 Wi-Fi 局域网。
- Windows 防火墙是否允许 Python/Uvicorn 监听 8080 端口。
- `SERVER_BASE_URL` 是否填的是 PC 的局域网 IP，而不是 `127.0.0.1`。

## 4. 配置并烧录固件

### 4.1 打开固件项目

用 VS Code 打开：

```text
E:\陈晨\FJ-ker\firmware
```

确保已安装 PlatformIO 扩展。PlatformIO 应识别到 `platformio.ini`。

### 4.2 配置 Wi-Fi 和服务端地址

复制固件密钥模板：

```powershell
cd E:\陈晨\FJ-ker\firmware
copy include\secrets.h.example include\secrets.h
notepad include\secrets.h
```

把 `include/secrets.h` 改成类似下面这样：

```cpp
#pragma once

#define WIFI_SSID "你的Wi-Fi名称"
#define WIFI_PASS "你的Wi-Fi密码"
#define SERVER_BASE_URL "http://192.168.1.42:8080"
```

注意：

- `WIFI_SSID` 和 `WIFI_PASS` 必须是 ESP32 能连接的 2.4 GHz Wi-Fi。
- `SERVER_BASE_URL` 必须是 PC 服务端打印出来的局域网地址。
- 不要填 `localhost` 或 `127.0.0.1`，因为那是 ESP32 自己，不是你的 PC。
- `secrets.h` 已被忽略，不应提交或分享。

### 4.3 烧录固件

1. 把 ESP32-S3-CAM 接到电脑。
2. 在 VS Code 左侧打开 PlatformIO。
3. 选择 `esp32-s3-devkitc-1` 环境。
4. 点击 Upload。

如果上传失败：

- 确认使用的是可烧录的 USB-C 口。
- 尝试按住 BOOT 键再点击 Upload。
- 检查串口是否被其他程序占用。
- 首次构建需要联网下载 `pioarduino/platform-espressif32` 和 `ArduinoJson`。

如果你熟悉命令行，也可以在 `firmware` 目录使用 PlatformIO CLI：

```powershell
cd E:\陈晨\FJ-ker\firmware
pio run
pio run -t upload
pio device monitor -b 115200
```

### 4.4 打开串口监视器

烧录后可以打开 PlatformIO Serial Monitor。

当前配置：

```ini
monitor_speed = 115200
```

如果运行异常，串口输出可以帮助判断是 Wi-Fi、摄像头、服务端还是显示屏问题。

## 5. 正常使用流程

### 5.1 开机

打开主电源开关。

正常现象：

- 2 秒内出现 `FJ-ker` 启动画面。
- 如果电池电压低于 3.3 V，会显示低电量提示并拒绝拍摄。

### 5.2 进入预览

按拍摄键（Capture）。

正常现象：

- 屏幕进入相机预览。
- 预览是单色抖动图，不是彩色图。
- 屏幕底部显示取景提示。

### 5.3 拍照并解题

对准题目后，按确认键（Confirm）。

设备会依次执行：

1. 切换到高分辨率 JPEG 拍摄。
2. 连接 Wi-Fi。
3. 通过 `POST /jobs` 上传照片。
4. 显示“正在思考”动画。
5. 每 1.5 秒轮询 `GET /jobs/{id}`。
6. 服务端完成后，下载答案页面。
7. 显示第一页答案。

正常耗时通常是 7 到 14 秒，取决于 Wi-Fi、照片大小和 DashScope 响应速度。

### 5.4 翻页

答案页中：

- 按 PageDown 到下一页。
- 按 PageUp 到上一页。
- 到第一页或最后一页时会停止，不会循环。

### 5.5 取消

不同状态下取消键（Cancel）的行为：

| 当前状态 | Cancel 行为 |
|---|---|
| 空闲 | 不做事 |
| 预览 | 留在预览 |
| 正在上传/正在思考 | 返回预览，并向服务端发送 `DELETE /jobs/{id}` |
| 答案视图 | 返回预览 |
| 低电量 | 不允许退出低电量锁定 |

### 5.6 再拍一道题

在答案视图按确认键（Confirm），会重新进入预览，然后可以继续拍下一题。

## 6. 服务端接口说明

固件和服务端使用 HTTP 轮询协议。

### 6.1 健康检查

```http
GET /health
```

返回：

```json
{"ok":true,"version":"0.1.0"}
```

### 6.2 上传题目照片

```http
POST /jobs
```

请求体：

- `multipart/form-data`
- 字段名：`image`
- 文件类型：JPEG

返回：

```json
{"job_id":"ab12","status":"queued"}
```

### 6.3 查询任务状态

```http
GET /jobs/{job_id}
```

可能返回：

```json
{"status":"queued","pages":0,"error":null}
```

```json
{"status":"processing","pages":0,"error":null}
```

```json
{"status":"ready","pages":3,"error":null}
```

```json
{"status":"error","pages":0,"error":"错误信息"}
```

### 6.4 获取某一页答案

```http
GET /jobs/{job_id}/pages/{index}
```

返回：

- `Content-Type: application/octet-stream`
- 正好 8064 字节
- 384×168，1-bpp，MSB-first，row-major
- 每行 48 字节，合计 168 行
- `1` 表示黑色墨点，`0` 表示白色

### 6.5 取消任务

```http
DELETE /jobs/{job_id}
```

用于用户在“正在思考”期间按取消键时，通知服务端停止浪费额度。

## 8. 常见问题排查

### 8.1 `install.ps1` 提示找不到 Python 3.11

检查：

```powershell
py -3.11 --version
```

如果失败，请安装 Python 3.11，并勾选 Python Launcher。

### 8.2 服务端启动后 ESP32 连接不上

检查：

- `run.ps1` 打印的 IP 是否填入了 `firmware/include/secrets.h`。
- ESP32 和 PC 是否在同一个局域网。
- PC 防火墙是否允许 8080 端口。
- `SERVER_BASE_URL` 是否形如 `http://192.168.x.x:8080`。
- 不要使用 `localhost`、`127.0.0.1` 或 PC 的公网 IP。

### 8.3 `/health` 打不开

检查：

- 服务端 PowerShell 窗口是否还在运行。
- 是否先运行过 `.\install.ps1`。
- 是否在 `server` 目录运行 `.\run.ps1`。
- 8080 端口是否被别的程序占用。

### 8.4 上传后一直“正在思考”

可能原因：

- DashScope API Key 错误。
- `QWEN_MODEL` 模型名不可用。
- PC 无法访问 DashScope 网络。
- DashScope 额度不足。
- 服务端报错但设备未及时刷新。

处理：

1. 查看运行 `.\run.ps1` 的 PowerShell 窗口是否有错误。
2. 打开 `.env` 检查 `DASHSCOPE_API_KEY`。
3. 如模型名被拒绝，尝试把 `QWEN_MODEL` 改成 DashScope 当前支持的视觉模型。

### 8.5 答案页为空或乱码

可能原因：

- Playwright Chromium 未安装成功。
- 字体缺失。
- 渲染失败后使用了简化 fallback。

处理：

```powershell
cd E:\陈晨\FJ-ker\server
.\.venv\Scripts\python.exe -m playwright install chromium
```

如需更好的中文显示，可把 `NotoSansSC-Regular.otf` 放到：

```text
server\app\fonts\NotoSansSC-Regular.otf
```

### 8.6 PlatformIO 烧录失败

检查：

- USB 线是否支持数据传输。
- 是否选中了正确串口。
- 串口是否被 Serial Monitor 占用。
- 是否需要按住 BOOT 再上传。
- 首次构建是否有网络访问权限。

### 8.7 显示屏无显示

检查：

- 显示屏 VCC 是否接 3.3 V，而不是 5 V。
- GND 是否共地。
- SCLK/MOSI/CS/DC/RST 是否接到正确 GPIO。
- ST7305 初始化序列可能需要根据供应商示例微调。

### 8.8 预览很慢或有残影

这是单色反射式 LCD 的硬件限制。固件已使用局部刷新接口降低残影，但实际效果仍取决于 ST7305 模块的驱动序列和屏幕刷新特性。

### 8.9 低电量时无法拍摄

这是正常保护行为。请充电后再使用。低于约 3.3 V 时，固件会锁定拍摄功能，避免锂电池过放。

## 9. 日常使用建议

- 每次使用前先启动 PC 服务端。
- 每次换 Wi-Fi 或 PC IP 改变后，都要更新固件中的 `SERVER_BASE_URL` 并重新烧录。
- 不要把 `.env`、`secrets.h`、DashScope API Key 或 Wi-Fi 密码发给别人。
- 如果只是测试服务端，不需要连接硬件，可以先运行服务端测试。
- 如果要长期使用，建议给 PC 设置固定局域网 IP，减少反复改 `SERVER_BASE_URL` 的麻烦。

## 11. 最小启动流程

如果你已经接好硬件，只想快速跑起来，按下面顺序做：

1. 在 PC 上进入服务端目录：

   ```powershell
   cd E:\陈晨\FJ-ker\server
   .\install.ps1
   copy .env.example .env
   notepad .env
   ```

2. 填入 `DASHSCOPE_API_KEY`。

3. 启动服务端：

   ```powershell
   .\run.ps1
   ```

4. 记下服务端打印的地址，例如：

   ```text
   http://192.168.1.42:8080
   ```

5. 配置固件：

   ```powershell
   cd E:\陈晨\FJ-ker\firmware
   copy include\secrets.h.example include\secrets.h
   notepad include\secrets.h
   ```

6. 填入 Wi-Fi 名称、Wi-Fi 密码、`SERVER_BASE_URL`。

7. 用 VS Code + PlatformIO 打开 `firmware/` 并点击 Upload。

8. 设备开机后：

   ```text
   Capture → 预览
   Confirm → 拍照上传
   等待“正在思考”
   PageUp/PageDown → 翻页
   Cancel → 返回预览
   ```
